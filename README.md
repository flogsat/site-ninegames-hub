# site-ninegames-hub

本仓库用于归档和发布多个独立的 HTML 页面。这些页面各自实现特定的功能或展示内容，彼此之间没有强关联，也不针对任何特定网站或域名。

## 项目简介

`site-ninegames-hub` 是一个 HTML 页面集合仓库，主要目的是将零散的页面资源进行集中管理和版本控制。每个页面均可独立运行，适合作为静态资源存档、功能演示或小型工具页面的托管库。

## 目录说明

```
.
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的公共资源（CSS、JS、图片等）
├── docs/           # 文档或说明文件（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或文件对应一个独立页面，命名尽量体现页面功能。
- `assets/` 用于存放多个页面可能共享的样式、脚本或媒体文件，避免重复。
- 页面间的资源引用建议使用相对路径，以保证在本地或静态服务器上均可正常访问。

## 页面归档说明

- 每个页面均为自包含的 HTML 文件，原则上不依赖外部服务或动态后端。
- 页面内容可能包含静态信息展示、交互组件或小型演示工具。
- 归档的页面可能来自不同时期或不同用途，更新频率不定。
- 不保证所有页面长期可用或与最新浏览器完全兼容，但会尽量保持基础功能稳定。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/site-ninegames-hub.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件即可查看对应页面。
3. 如需部署到静态服务器，可将整个仓库内容上传至服务器根目录，确保文件路径结构与本地一致。
